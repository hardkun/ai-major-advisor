# 项目不足与后续优化

## 当前不足

1. 当前数据不是全量高校数据，只是部分高校公开数据验证集。
2. 高校官网结构差异很大，自动采集准确率有限，仍需要人工核验。
3. 搜索结果可能混入硕士、博士、招生计划、新闻稿或第三方转载页面。
4. 当前没有接入真实微信支付，只实现了 mock-pay 演示流程。
5. 当前 admin 后台没有权限系统，仅适合本地作品集演示。
6. 推荐算法第一版偏规则，没有引入复杂预测模型或个性化模型。
7. 当前部署和小程序上线还需要合规审核、域名配置、隐私政策和权限控制。

## 后续优化

1. 引入更稳定的官方数据源，或建立人工审核后的标准数据集。
2. 建立更严格的本科招生页面识别规则，减少硕博页面、招生章程和计划页面混入。
3. 增加数据版本管理，记录不同年份、批次、来源和导入批次。
4. 增加管理员登录、权限分级和操作审计。
5. 增加线上部署、日志监控、异常告警和定期备份。
6. 扩展更多省份、年份和科类的数据验证集。
7. 将推荐解释进一步结构化，例如学习能力要求、课程模块和职业方向标签。
8. 增加用户反馈闭环，用于优化方向标签、解释质量和推荐规则。

## 收尾判断

作为求职作品集，本项目已经具备完整闭环：数据采集 → raw 入库 → 人工核验 → 正式推荐 → AI 解释 → 报告展示 → 后台治理。

后续不应继续盲目追求全量高校覆盖，而应围绕稳定演示、文档表达、代码可运行性和项目边界说明进行打磨。
