# 岗位匹配说明

## 1. AI应用开发工程师

匹配点：

- LLM API 接入。
- 推荐结果 AI 解释生成。
- Prompt 和结构化 JSON 输出。
- 将 AI 能力嵌入真实业务流程。
- 在高责任场景中限制 AI 决策边界。

可强调：这个项目不是聊天机器人，而是把 LLM 放在推荐结果解释层，结合规则、数据和风险提示形成可控 AI 应用。

## 2. Python后端开发

匹配点：

- FastAPI 后端开发。
- REST API 设计。
- SQLite 数据库设计。
- routers / schemas / crud / services 分层。
- admin 后台和脚本工具。
- 错误处理、数据库迁移和项目健康检查。

可强调：项目包含推荐、报告、采集、核验、数据源管理等多个业务模块，不是单表 CRUD。

## 3. 大模型应用开发

匹配点：

- 基于已有推荐结果生成 AI 解释。
- 将风险提示写入 AI 输出。
- AI 不直接决定推荐结果，降低幻觉影响。
- 支持 fallback，AI 调用失败不影响主业务。
- 通过结构化 schema 约束模型输出。

可强调：重点是“大模型如何安全进入业务系统”，而不是简单调用模型。

## 4. 数据工程 / 数据采集方向

匹配点：

- 数据源发现和搜索。
- HTML / Excel / PDF 解析。
- 字段映射、表头识别、宽表转换。
- raw/admissions 双层数据。
- 覆盖率报告和缺口诊断。
- 数据质量检查和人工核验。

可强调：项目围绕公开招生数据构建了小型数据管线，并考虑了来源追溯和质量控制。

## 5. FDE / 行业解决方案预备方向

匹配点：

- 面向真实业务问题建模。
- 能把用户流程、后台流程和数据流程串起来。
- 设计人机协同审核闭环。
- 输出可解释结果和风险提示。
- 能对项目边界、数据限制和后续规划做清晰说明。

可强调：项目展示的不只是代码能力，还包括需求拆解、产品边界、数据治理和面向演示交付的能力。
